# CLAUDE.md

## Idioma

- Responda em português brasileiro (pt-BR).
- Mantenha explicações claras e objetivas.

## Sobre o projeto

Antes de modificar qualquer arquivo:

- Entenda a estrutura existente.
- Analise como os componentes funcionam.
- Evite alterações desnecessárias.

## Código

Priorize:

- código limpo e legível;
- componentes reutilizáveis;
- organização simples;
- boas práticas da linguagem utilizada.

Evite:

- duplicação de código;
- complexidade sem necessidade;
- alterações que quebrem funcionalidades existentes.

## Alterações

Antes de implementar:

1. Entenda o problema.
2. Identifique a causa.
3. Faça a menor alteração necessária.
4. Verifique possíveis impactos.

## Frontend

Ao trabalhar na interface:

- mantenha consistência visual;
- preserve responsividade;
- reutilize componentes existentes;
- considere acessibilidade e experiência do usuário.

## Bugs

Ao corrigir problemas:

- procure a causa raiz;
- explique o motivo do problema;
- evite soluções temporárias.

## Dependências

Antes de adicionar bibliotecas:

- verifique se já existe uma solução no projeto;
- avalie impacto e necessidade.

## Segurança

Nunca exponha:

- tokens;
- senhas;
- chaves privadas;
- informações sensíveis.

## Objetivo

Toda alteração deve melhorar o projeto sem adicionar complexidade desnecessária.
